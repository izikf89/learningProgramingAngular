הקדמה

המתודה constructor מאפשרת הזרקה של service על ידי רישום כפרמטר את הקלאס שמייצג את הservice, כמו בדוגמא: {{ injectConstructor }},
ניתן להזריק במתודה constructor בצורה רגילה רק service שמיוצגים על ידי קלאס ולא שום type אחר,

Injectable@

כל קלאס ואפילו סתם משתנה יכול להיות service גם בלי שיש לפניו את המתודה @Injectable,
המתודה @Injectable נצרכת רק מתי שיש בתוך הservice שימוש לייבא קלאסים אחרים באמצעות המתודה constructor,
דוגמא לservice שמשתף מידע בין component אפילו שאין לו את המתודה @Injectable:

שתי מילים על Injector: Injector מטפל בservice, הוא מחזיק שני מתודות: create בשביל לרשום וget בשביל להזריק

Injector.get

Injector.get מאפשר לגשת לservice בתוך component גם בלי להזריק אותו במתודה constructor,

1) יש לייבא את Injector מהספריה @angular/core, דוגמא: {{ importInjector }},
2) רושמים את הInjector במתודה constructor, דוגמא: {{ constructorInjector }},
3) בInjector משתמשים במתודה get שמקבלת כפרמטר את הservice שרושמים ומכניסים את מה שמתודה מחזירה לvariable, דוגמא: {{ Injector }},

Injector יכול להזריק גם service שמיוצגים לא על ידי קלאס כמו בדוגמא הבאה שהservice מיוצג על ידי string

Injector.create

מאפשר לרשום service מחוץ לcomponent או מודול

1) יש לייבא את Injector מהספריה @angular/core, דוגמא: {{ importInjector }},
2) יש להוסיף את הproviders בתוך המתודה create כמו שרושמים בcomponent או מודול, המתודה תחזיר injector שמחזיק את הservice, דוגמא: {{ injectorCreate }}
3) בשביל לגשת לservice יש להשתמש במתודה get שמחזירה הזרקה של הservice כמו בדוגמא: {{ InjectorGetService }}

Inject@

המתודה Inject מאפשר להזריק במתודה constructor service שאינם מיוצגים על ידי קלאס,

1) יש לייבא את Inject מהספריה @angular/core, דוגמא: {{ importInject }}, 2) את המתודה Inject מכניסים כפרמטר למתודה constructor, המתודה Inject מקבלץ כפרמטר את הservice ומוסיפים אחרי המתודה עוד מילה שתייצג את הservice בcomponent

InjectionToken

מיועד ליצור token שאינו קלאס,

1) יש לייבא את InjectionToken מהספריה @angular/core, דוגמא: {{ importInjectionToken }},
2) ליצור את הtoken כמו בדוגמא: {{ InjectionToken }},

חובה לשים בInjectionToken פרמטר string, ואין שום משמעות מה הstring ואפשר לכתוב שם מה שרוצים או בכלל לא כמו בדוגמא הנ"ל,
אין כמעט הבדל אם עושים token עם InjectionToken או סתם string, יש הבדל אם יש שני token שהערכים שלהם שווים כמו בדוגמא שאפשר לראות בקישורים הבאים:
בגלל שאני לא רואה צורך להרחיב בנושא של InjectionToken ולכתוב על זה פוסט שלם אני יעצור פה, מי שמעוניין לראות יותר יוכל למצוא מידע בפוסט הזה

Optional@

אם ננסה להזריק במתודה constructor service שלא נרשם בproviders התוכנית תעצר, Optional גורם שהתוכנית לא תעצר,

1) יש לייבא את Optional מהספריה @angular/core, דוגמא: {{ importOptional }},
במתודה constructor לפני ששמים את הservice יש לשים את המתודה Optional כמו בדוגמא: {{ optional }}

בצורה רגילה כשמזריקים service במתודה constructor המתודה בודקת אם הcomponent שמארח אותה רשם את הservice, במידה ולא היא בודקת האם הservice רשום בcomponent שמארח אותה וכן הלאה, המתודה תמיד תחפש במעלה העץ של הקלאסים האם אחת מהקלאסים רשמה את הservice, המתודות שנראה בהמשך הפרק משנות את צורת החיפוש של המתודה,

SkipSelf@

אומר למתודה constructor לא לחפש בcomponent הנוכחי אלא להתחיל לחפש בparent שלו,

1) יש לייבא את SkipSelf מהספריה @angular/core, דוגמא: {{ importSkipSelf }},
במתודה constructor לפני ששמים את הservice יש לשים את המתודה SkipSelf כמו בדוגמא: {{ importSkipSelf }}

Self@

אומר למתודה constructor לחפש רישום של service אך ורק בתוך הcomponent שבו היא נמצאת,

1) יש לייבא את Self מהספריה @angular/core, דוגמא: {{ importSelf }},
במתודה constructor לפני ששמים את הservice יש לשים את המתודה Self כמו בדוגמא: {{ Self }}

Host@

אומר למתודה constructor לחפש רישום של service אך ורק בתוך הelement שבו היא נמצאת,

1) יש לייבא את Host מהספריה @angular/core, דוגמא: {{ importHost }},
במתודה constructor לפני ששמים את הservice יש לשים את המתודה Host כמו בדוגמא: {{ Host }}
