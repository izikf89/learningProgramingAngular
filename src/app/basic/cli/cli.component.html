הcli מופעל משורת הפקודה בלבד,
כל פקודה של הcli מתחילה ב ng ,
חוץ מיצירת פרוייקט כל הפקודות חייבות לפעול כשאתם עומדים בתיקיית הפרוייקט (root folder).

יצירת פרוייקט

הקלידו ng new anglar5 זה ייצור תיקייה עם השם angular5 שבתוכה יש פרוייקט של אנגולר, ההתקנה יכולה לקחת הרבה זמן כיון שה npm מתקין הר של ספריות שקשורות לאנגולר.

נווטו לתיקיית הפרוייקט רק משם תוכלו להפעיל את ה cli בפרוייקט שלכם,

להלן רשימה של הפקודות הנפוצות של הcli:

יצירת קבצים

הcli יוצר עבורנו: א: קובץ עם הקלאס שביקשנו ממנו והקוד המינמלי ההכרחי, ב: יוצר עבור הקובץ תיקייה חדשה שבא הקובץ ועם קבצים אחרים שהוא צריך בcomponent לדוגמא הcli יוצר קבצי html css וקובץ של טסט, ג: הcli רושם את הקלאס שנוצר בקלאס module,

ng generate type fileName --routing -flat -it -is
generate - אפשר לכתוב בקיצור g, נראה כך: ng g component name.
type - פה קובעים את סוג הקלאס שאנחנו מעוניינים בו, סוגי הקלאס הם: class, component, directive, enum, guard, interface, module, pipe, service, אפשר לכתוב בקיצור את האות הראשונה של השם של הקלאס, לדוגמא בשביל ליצור component אפשר לכתוב כך: ng generate c name.
name - פה כותבים את השם שבא לכם.
--routing - אופציונלי, לקבצים מסוג module בלבד, מגדיר ראוטר ויוצר קובץ מודול נוסף שבו נרשם הראוטר, אפשר לבטל את היצירה של המודול הנוסף ולרשום במודול שכבר קיים על ידי הוספה -flat
--flat - אופציונלי, עושה שלא תיווצר תיקיה חדשה אלא הקבצים יתווספו בתיקיית היעד.
-is - אופציונלי, ל component בלבד, עושה שלא יווצר קובץ style.
-it - אופציונלי, ל component בלבד, עושה שלא יווצר קובץ html.

ניתן ליצור תיקיות או להכניס לתיקייה קיימת על ידי כתיבת שם התיקיה וסלאש לפני שם הקבץ כמו זה: ng g c folderName/fileName .

בניית הפרוייקט

בשביל לעלות את האתר לשרת חובה לבנות את הפרוייקט על ידי הפקודה build, הפקודה הזאת יוצרת תיקייה חדשה בתיקיית הפרוייקט שנקראת dist, בתוכה נמצאים קבצי ה index.html cssו js שבתוכם מקומפל כל הקוד שכתבנו ואותם מעלים לשרת

ng build [--prod] [-bh=url]
--prod - מכווץ את הקבצים הרבה יותר אבל גם לוקח יותר זמן , בשביל שרת פרודקשן זה חובה בגלל שב build רגיל הקבצים גדולים מאוד,
-bh - אופציונלי, מוסיף url לתג base, זה הכרחי כשמעלים לשרת כדי שידע איפה תיקיית התוכנה ממוקמת ביחס לשרת,

שרת לוקאלי

אחד מהתוספות החזקות של אנגולר, הפעלת האופציה הזאת תעלה לכם שרת לוקאלי (localhost:4200), הcli יעשה build ויעלה באופן אוטומטי את הפרוייקט שלכם בכל פעם שתשנו קובץ בפרוייקט ותשמרו אותו,
זה כלי קריטי לפיתוח של האתר מכיון שהוא מאפשר לכם לראות כל רגע איך האתר שבניתם נראה ועובד בפועל.

ng serve -o -p 4201 [
serve - אפשר גם בקיצור s, נראה כך: ng s,
-o - אופציונלי, פותח אוטומטית את השרת בדפדפן כשהשרת גומר לעלות,
-p 4200 - אופציונלי, מגדיר באיזת פורט יעלה השרת

עידכון ספריות קיימות

ng update libraryName
בשביל לעדכן סיפריות בלי לחשוש לקונפליקטים בין העידכונים עדכנו דרך הפקודה הזאת ולא דרך npm,

הוספת ספריות

ng add libraryName
בשביל להוסיף ספריות בלי לחשוש לקונפליקטים בין הגירסאות של הסיפריות הוסיפו את הסיפריה דרך הפקודה הזאת ולא דרך npm,

יש עוד אופציות רבות שאפשר לראות כאן.
