במידה ואתם יודעים להשתמש ב command line עברו למדריך הבא.

פתיחה

איך נכנסים: לחצו על מקש חלונות + S , בתיבת חיפוש שנפתחה הקלידו command prompt או powerShell וכנסו לתוצאה,
בחלון שנפתח כותבים פקודות ומפעילים אותם על ידי לחיצה על enter,

מה זה command line

כשאנחנו מבצעים פעולה במחשב כמו פתיחת תיקייה או פתיחת של תוכנה, מחיקה העתקה ויצירה של קבצים וכו' אנחנו בעצם מפעילים פקודה שהמחשב יודע לתרגם לפעולה בזיכרון ולהראות את התוצאה על המסך,
את הפקודות האלה עושים בדרך כלל בעכבר , בשורת הפקודה (באנגלית command line) אפשר להפעיל את כל הפקודות בשורת הפקודה,
לדוגמא כיתבו את הקוד הבא ולחצו enter: explorer = התוצאה זה שהמחשב שלי נפתח,
תוכנה למשתמש רגיל עושה שימוש בממשק ויזואלי, בתיכנות תוכנות רבות משתמשות בממשק של שורת הפקודה,
שורת הפקודה מכונה cli, command line, cmd, ו shell, באתר נשתמש בכינוי cli.

שימוש בסיסי

ניווט: כמו שבחלונות יש ניווט מתיקייה לתיקייה כך יש גם ב cli ,
הניווט מתבצע של ידי המילה cd ואחריו את הכתובת הרצויה, לדוגמא cd c:\users,
כדי לראות מה הכתובת של חלון אפשר ללחוץ עם העכבר פעם אחת על שורת הכתובת,
כשמפעילים פקודה של תוכנה הפקודה תמיד תתחיל בשם של התוכנה, לדוגמא פקודה בnpm נראית כך: npm install something.

איפה מפעילים פקודות


יש פקודות בסיסיות של המחשב שהcli יבצע בכל כתובת כמו cd, פקודות שמתוכנתות על ידי קבצים אפשר להתקין אותם בשני צורות: מקומי (localy) או גלובאלי (globaly), ההבדל שאם ההתקנה היא מקומית אז הפקודות יעבדו רק בתיקיה שבא נמצא הקובץ שמגדיר את הפקודות, ואם ההתקנה היא גלובאלית אז אפשר להפעיל את הפקודות מכל מקום במחשב,
root folder (תיקיית הפרוייקט): גם אם ההתקנה היא גלובאלית יש תוכנות שהפקודות שלהם יעבדו רק כאשר נמצאים בתוך התיקייה הראשית של פרוייקט של התוכנה, תיקיית הפרוייקט נקראת root folder,
בangukar cli הפקודה שיוצרת פרוייקטים תעבוד מכל כתובת במחשב אבל שאר הפקודות יעבדו רק מהroot folder,

npm

npm זה תוכנה שבעזרתה מתקינים מעדכנים ומוחקים תוכנות וספריות קוד,
פקודה של התקנה מתחילה בשם התוכנה npm וממשיכה עם המילה install, אפשר להוסיף -g בשביל שההתקנה תיהיה גלובאלית, ולאחר מכן כותבים את השם של התוכנה,
לדוגמא התקנה של angular cli נראית כך: npm install -g @angular/cli,
מחיקה של angular cli נראית כך: npm remove -g @angular/cli.
