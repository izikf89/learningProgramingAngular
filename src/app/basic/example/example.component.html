בפרק זה מובא הדגמה מינמלית של אתר אנגולר והסבר על קובץ מודול וcomponent,

יצירה

1) צור פרוייקט בcli על ידי הפקודה: ng new hello (זה יכול לקחת 10 דקות), לאחר מכן נווט לתוך התיקייה על ידי הפקודה cd hello,
2) הפעל את השרת על ידי הפקודה ng s -o, כעת יפתח הדפדפן ויעלה האתר,
האתר אמור להראות כך:

הסבר

קלאסים מסוג component הם למעשה החלק העיקרי של האתר, בהם יש את הHTML של האתר,
component הוא למעשה קלאס שהוגדר על ידי decorated @Component,
component יוצר תג html חדש שמכיל html, css וtypescript, הסבר כיצד מגדירים:

  • שם התג שנוצר מוגדר בproperty selector כמו זה: במקרה הזה הוגדר תג בשם app-root,
  • הtypescript נמצא בתוך הקלאס כך:
  • הHTML והcss נמצאים בproperties של המתודה Component@,
    הHTML והcss יכולים ליכתב ישירות לproperties כך: הproperties יכולים להכיל במקום זאת קישור לקובץ html או css כך:

המעלה הגדולה של component זה שכל component חי לעצמו ואין לcomponent אחרים השפעה עליו בtypescript והcss שלהם, מה שמאפשר לבנות את האתר כאילו הוא עשוי ממספר אתרים נפרדים באותו עמוד,
כשהcli יוצר component הוא יוצר איתו קבצי html css וקובץ טסט,

קובץ מודול

בתיקיית app כנסו לקובץ מודול (app.module.ts), הוא צריך להראות כך:

הסבר

קובץ מודול מכיל את כל הקלאסים שקיימים בפרוייקט (יכולים להיות כמה מודולים), קלאס שלא נרשם במודול לא יעבוד,
קובץ מודול מכיל מספר properties שכל אחד מהם מכיל מערך של קלאסים,
הקלאסים שנרשמים במודול חייבים להיות מיובאים לקובץ שבו נמצא הקלאס מודול על ידי import,
כל סוג קלאס צריך להירשם בproperty המתאים, פירוט:

  • declartions - מכיל את כל הקלאסים מסוג component directive ו pipe,
    אפשר לרשום קלאס מהסוג הנ"ל בקובץ מודול אחד בלבד, כדי שיהיה אפשר להשתמש בו גם במודולים אחרים צריך לרשום אותו גם בexports,
  • imports - מכיל קלאסים מסוג מודול בלבד,
    בתוך המודול שבו נרשם ניתן להשתמש בכל הקלאסים שנרשמו במודול הנרשם,
  • providers - מכיל את כל הקלאסים מסוג service,
    רישום של service יוצר העתק חדש של ה service לשימוש המודול שבו הוא נרשם בלבד,
  • exports - מכיל קלאסים שנרשמו כבר במודול,
    קלאסים שרשומים פה יהיו זמינים גם במודולים אחרים,
  • bootstrap - מכיל קלאסים מסוג component בלבד,
    ה property הזה מופיע אך ורק במודול הראשי של הפרוייקט,
    הקלאסים שרשומים פה הם אלה שיתחילו לעבוד כשהאתר יעלה,
    component שרשומים בbootstrap התג שלהם צריך להיות בקובץ index.html,
