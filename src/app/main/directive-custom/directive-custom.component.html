מה זה עושה

directive נועד בשביל ליצור attributes חדשים שמבצעים פעולות על התג המארח,

בחלק מהדוגמאות נעשה שימוש במתודה ngOnInit, המתודה הזאת מובנית באנגולר ומוגדרת שהיא מופעלת באופן אוטומטי כשהקלאס נוצר,

איך כותבים

1) יוצרים קלאס מסוג directive (בcli ng g d myDirectiveName),
2) מגדירים את השם של הdirective החדש בתוך סוגריים מרובעות בproprty selector שנמצא בקלאס @Directive כמו בדוגמא: {{ selectorExample }},
3) כותבים את הפעולות שאנחנו מעוניינים לבצע עלוי בקלאס של הdirective,
4) רושמים את הdirective במודול בproprty declarations,

כעת נפרט את הפעולות שאפשר לבצע בdirective:

properties

ניתן לגשת לproperties של התג המארח,
הגישה נעשית באמצעות הservice ElementRef והproperty שלו nativeElement,

event

ניתן להאזין לevent שמתרחש בתג באמצעות המתודה HostListner,

input

ניתן ליצור attribute נוסף שיקבל ערך עם המתודה Input,

ניתן להגדיר שהdirective עצמו יקבל ערך,

הערה: אם שמתם לב שהinput צריך להיות בסוגריים מרובעים מכיון שהוא מתקשר בין התצוגה והקוד.

selector

עד עכשיו בproperty selector הגדירנו את הdirective כattribut אחד בלבד, ישנם עוד אפשרויות הגדרה בselector כדלהלן:

ניתן להגדיר את הdirective כקלאס css או תג html (id אי אפשר) וכן ניתן להגדיר את הdirective בכמה כינויים כמו בדוגמא הבאה: selector: '.directiveClass, directiveTag', ,
דוגמא לקובץ מודול: דוגמא לcomponent:

ניתן להגדיר שהdirective יעבוד רק אם יקבל ערך כמו בדוגמא הבאה: selector: '[attribute=something]',

ניתן להגדיר שהdirective יפעל רק בתג מסוים כמו בדוגמא הבאה: selector: 'tagName[attribute]',

ניתן להגדיר directive גם על תג או attribute שכבר קיימים כמו לדוגמא התג h1 והattribute type,

ניתן להגדיר שהdirective יחול בכל מקום חוץ מאשר בתג או cssClass כמו בדוגמא הבאה: selector: ':not(h1)',
