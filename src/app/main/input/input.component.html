לתג input יש כמה כלים שמיוחדים בשבילו

בשביל הכלים של input יש צורך לייבא מודול בשם FormsModule דוגמא: {{ import }}
כעת יש לרשום אותו בקלאס מודול בproperty שנקרא imports,

ngModel

הdirective ngModel יוצר binding לinput לשני הכיוונים מהתצוגה לקוד וגם להפך,
כותבים את זה כך: {{ ngModel }}

select

הdirective ngModel יכול להיות גם בתג select,
כותבים את זה כך: {{ select }}

radio

הdirective ngModel יכול להיות גם בשדה מסוג radio,
כותבים את זה כך: {{ radio }}

checkbox

הdirective ngModel יכול להיות גם בשדה מסוג checkbox,
כותבים את זה כך: {{ checkbox }}

ngValue

ngValue עושה שngModel בselect יקבל את כל האובייקט הנבחר,
הסבר: בselect ngModel מקבל את הערך שנכתב בתוך הסוגריים המסולסלות, לכן כשהngFor מקבל מערך של אובייקטים ובתוך הסוגריים המסולסלות פונים לkey מסויים, אז התוצאה תיהיה שהngModel יקבל רק את הvalue של הkey ולא את כל האובייקט עצמו, הngValue גורם שנקבל את כל האובייקט,
הפעלה: מוסיפים את הdirective ngValue ומכניסים לngValue את הindex, כמו בדוגמא הבאה: {{ ngValue }}

ngModelChange

ngModelChange זה event שמופעל כל פעם שיש שינוי בngModel,
בשביל לגשת לערך החדש מוסיפים את הngModelChange כמו כל event (ngModelChange)="onChange($event)"

validation

אין פה משהו חדש רק לתשומת ליבכם באנגולר בשביל להציג הודעות שגיאה משתמשים בhashtag, מכיון שhashtag יכול לגשת לproperties ובproperties נמצאים הvalidation

validation class

כשכותבים ngModel בinput אנגולר מוסיף באופן אוטומטי class לפי מצב הvalidation, לדוגמא הclass ng-touched מופיע אחרי שהמשתמש נגע בinput ועזב,

להלן רשימה של הclass שאנגולר מוסיף עם הסבר מתי מופיע:

  • ng-touched - מופיע כשהמשתמש נגע ועזב
  • ng-untouched- מופיע כשהמשתמש לא נגע ועזב
  • ng-dirty - מופיע כשהמשתמש משנה את הערך בinput
  • ng-pristine - מופיע כל עוד שהמשתמש לא משנה את הערך בinput
  • ng-valid - מופיע כשכל הvalidation הם true
  • ng-invalid - מופיע כשאחד הvalidation לא true
