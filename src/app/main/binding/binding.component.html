מה זה

binding זה היכולת של אנגולר לתקשר בין הקוד (typescript) לתצוגה (html),
בצורה רגילה מעבירים מידע מהקוד לתצוגה ולהפך באמצעות getElementById('myId').innerHTML, באנגולר כל property שקיים בקלאס של הcomponent זמין כמשתנה בתצוגה,

פרטים

בhtml ניתן לגשת אל הproperty בתוך סוגריים עגולות או מרובעות או מסולסלות זה תלוי איפה ניגשים לproperty, ישנם ארבעה סוגי binding :

  • מהקוד לתצוגה - סוגריים מסולסלות, לדוגמא בקוד: {{ example6 }} בתצוגה: {{ example7 }},
  • מהקוד לattribute - הattribute צריך להיות בתוך סוגריים מרובעות (ניתן להכניס ערך לכל property של התג), לדוגמא בקוד: {{ example8 }} בתצוגה: {{ example9 }},
  • מattribute לקוד - סוגריים עגולות, לדוגמא בקוד: {{ example10 }} בתצוגה: {{ example11 }}, בדוגמא הזאת אנגולר ניגש למתודה בקוד כשקורה event מסוג click,
  • מהקוד לתצוגה וגם מהתצוגה לקוד - סוגריים מרובעות ובתוכם סוגריים עגולות, קיים רק במקרה שמשתמשים בngModel , זה לא יפורט בפרק זה,

כללים

  • ניתן גם לגשת למתודות שמחזירות ערך, לדוגמא בקוד יש מתודה: {{ example7 }} ניתן לקבל את הערך של המתודה על ידי שנכתוב בתצוגה {{ example3 }},

ההבדל ביןattribute לproperty

בתג html יש attribute בלבד, דוגמא לattribute זה id, לכל תג html הdom (שזה אפשר לומר המנוע של הjavascript) יוצר object שמשקף את התג html, הproperties של האובייקט הזה זה הattribute של התג ועוד כל הדברים שמגדירים את התג כמו הvalue שלו שהם אינם attribute,
כשמשנים את הproperties של האובייקט הdom משנה את התג בהתאם לשינויים שבוצעו באובייקט, באנגולר כשמשתמשים בbinding בשביל לשנות תג אז משנים את הproperties של התג והdom מעדכן את תג,
