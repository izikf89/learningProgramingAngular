pipe זה מתודה שפועלת על ערכים בתצוגה

pipe נראה כך: {{ pipeSyntex }},
pipe עובד כמו מתודה , היא לוקחת את הערך מבצעת עליו פעולות ומחזירה את התוצאה,
pipe יכול לקבל פרמטר, כמו בדוגמא: {{ parameter }},
אפשר לבצע כמה pipe על אותו ערך כמו בדוגמא: {{ severalPipe }},
יש pipe מובנים built-in באנגולר, ואפשר גם ליצור חדשים,

pipe built-in

נביא דוגמא של pipe שהופך אותיות גדולות לקטנות:

לפניכם רשימה של הpipe built-in העיקריים:

  • decimal - מטפל במספרים עשרוניים,
  • percent - הופך מספרים לאחוזים,
  • currency - הופך מספרים לפורמט של כסף
  • date - מציג אובייקטים של datetime בפורמטים שונים
  • json - מציג אובייקטים מסוג json,
  • slice - חותך string ומערכים,
  • titlecase - משנה בstring את האות הראשונה לאות גדולה,
  • lowercase - משנה את האותיות לקטנות,
  • uppercase - משנה את האותיות לגדולות,

חמש הpipe הראשונים ברשימה מקבלים פרמטרים ויש להם הרבה פורמטים שונים, פרטים ראו באתר של אנגולר

ליצור pipe

1) ייבא את המתודה decorated Pipe כמו בדוגמא: {{ importPipe }},
2) להגדיר את השם של הpipe כמו בדוגמא: {{ pipeName }},
3) בתוך הקלאס של הpipe הוסיפו את המתודה transform עם פרמטר אחד, הפרמטר זה הערך בתצוגה שעליו מבוצע הpipe,
4) לרשום את הpipe במודול בdeclartions property,

דרך הcli הנ"ל נוצרים אוטומטית הקלידו את הפקודה הבאה: ng g p pipe-name

בדוגמא הבאה נציג pipe שמכפיל את הערך בתצוגה כפול 2:

pipe בngFor

ניתן לשלב pipe בngFor כך שיובצע על כל ערך במערך,
בתצוגה זה צריך להראות כך: {{ ngFor }},
ובקוד של הpipe צריך לעשות לולאה על הפרמטר שבתוכה נעשים הפעולות על התאים שבמערך כמו בדוגמא הבאה:

pipe binding

pipe יופעל כל פעם שיש שינוי בערך שניתן לו, כמו שלדוגמא אם נשנה את הערך בקוד הpipe יבוצע עליו שוב כמו בדוגמא הבאה:

pure

pipe בצורה הרגילה לא יכול לזהת refernce אלא רק primitive זה אומר שהוא יזהה שינוי בstring number וboolean אבל לא object,
הברירת מחדל בpipe זה שהוא מופעל כל פעם שהערך שניתן לו משתנה כמו בדוגמא למעלה, אפשר להגדיר שהpipe יופעל כל כמה מילי שניות וכך לפתור את הבעיה של הזיהוי עם הrefernce, המצב הזה נקרא impure,
ההגדרה נעשית על ידי הוספת property בשם pure עם הערך false למתודה pipe כמו בדוגמא הבאה: {{ pure }},
